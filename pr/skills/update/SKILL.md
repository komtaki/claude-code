# PR 本文更新手順

日本語でのコミュニケーションを基本とする。

## PR の本文の重要な注意事項

- **create スキル**の「PR の本文の重要な注意事項」と「PR本文のエスケープについて」に従う

## 更新手順

1. 現在のブランチのPR情報を取得する (`gh pr view`)
2. ベースブランチとの実装差分を確認する (`git diff <base-branch>...HEAD`)
3. 既存のPR本文と実装差分を比較し、本文に反映すべき変更点を特定する
4. `gh pr edit --body` で本文を更新する
