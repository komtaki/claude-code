# PR 作成手順

日本語でのコミュニケーションを基本とする。

## commit

commit のフォーマット・粒度・Type は **commit スキル**に従う。

## PR の本文の重要な注意事項

- PR 作成時は必ず **@.github/pull_request_template.md** のテンプレートに従って body を記述する
- Howがある場合、なぜその実装になったか背景を書く。コードの変更行や内容をそのまま並べるだけの説明は書かない。
- 参照したドキュメントの URL があれば必ず PR の本文に記載
- Assign は komtaki を指定する
- テンプレートがない場合には以下のフォーマットに従う：
  - What
  - How
- PR の最後に、「Generated with AI」を追加

### PR本文のエスケープについて

- **シングルクォートで囲み、本文を直接記述する**: `gh pr create --body '本文をここに直接書く'`
- シングルクォート内では `$()` や HEREDOC は展開されずリテラル文字列になるため、**絶対に使わない**
- シングルクォート内ではバッククオートのエスケープは不要
- シングルクォート自体を含める場合は `'\''` で表現

## 作成手順

1. 現在のブランチとコミット状況を確認する
2. デフォルトブランチにいる場合は、fetchして最新にする。そのあと差分から名前を決めて新しくブランチを作成する。
3. ブランチを作成したら、ブランチを remote に push (git push)
4. **PR をドラフト状態で作成する** (`gh pr create --draft`)
5. CIの成功をサブエージェントで待機して、確認。CIが失敗したら修正する
