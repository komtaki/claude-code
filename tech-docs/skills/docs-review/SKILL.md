# 文章レビュー（サブエージェント起動）

## 用途

文章（技術ドキュメント、PR タイトル/本文、コミットメッセージ、Linear 起票文など）の執筆完了後に、独立した視点で冗長な記述・曖昧表現・事実誤認・表記揺れを検出する。自分で書いた文章は思い込みで読み飛ばすため、最終確認はサブエージェントに委ねる。

## 起動手順

`Agent` ツールで `general-purpose` サブエージェントを1つ起動し、対象テキストを冒頭から末尾まで通読させる。レビュー基準は `tech-docs:docs-write` の3原則（Correct / Clear / Concise）、執筆ルール、推敲チェックリスト。

呼び出し元ごとに対象の渡し方が異なる。

- ファイル: 絶対パスを渡し、サブエージェント側で `Read` させる
- PR タイトル/本文: `gh pr view <PR番号> --json title,body` の出力を本文に貼る
- コミットメッセージ: 候補メッセージ本文をそのまま貼る

## プロンプト雛形

````
次のテキストを `tech-docs:docs-write` スキルのルールに従ってレビューしてください。

対象種別: <ファイル / PR / コミットメッセージ / Linear起票文 / ...>
対象: <絶対パス または 本文を直接貼り付け>

レビュー観点:
- Correct: 事実誤認、固有名詞・数値・コマンドの誤り、変更内容との不一致
- Clear: 主語述語の不整合、指示代名詞の曖昧さ、専門用語の説明不足、セクション間の説明重複
- Concise: 冗長な補助動詞、「という」「のこと」、不要な箇条書き、コード差分の単純な書き写し
- 表記統一: 用語・送り仮名・記号・数字の揺れ

報告形式（指摘ごとに繰り返し）:
- 場所: <ファイル名:行番号 または セクション名 / タイトル>
- 引用: <該当箇所の原文>
- 分類: Correct / Clear / Concise / 表記
- 修正案: <書き換え後の文>

修正は提案のみで、対象テキスト・ファイル・PR 本体は編集しないでください。
````

## レビュー結果の反映

サブエージェントの指摘は鵜呑みにせず、`tech-docs:docs-write` のルールに照らして採否を判断する。採用した修正のみ反映し、却下した指摘は理由を添えてユーザーに共有する。反映先は呼び出し元スキルが指示する手段に従う（`Edit` / `gh pr edit` / `git commit --amend` など）。
